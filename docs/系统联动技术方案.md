## 3. 认证系统

### 3.1 企业用户登录

```
POST /api/auth/login
```

**Request:**
```json
{
  "countryCode": "+852",
  "phone": "63123456",
  "password": "Cainiao@2025"
}
```

**后端逻辑：**
1. 查 enterprises 表，匹配 country_code + phone
2. bcrypt 验证密码
3. 生成 JWT（payload 包含 enterprise_id, market, premium_rate, currency）
4. 设置 HttpOnly Cookie

**Response:**
```json
{
  "token": "eyJ...",
  "enterprise": {
    "id": "ent-001",
    "name": "菜鸟速递",
    "market": "HK",
    "currency": "HK$",
    "premiumRate": 1.15,
    "creditRemaining": 37500
  }
}
```

### 3.2 鉴权中间件

所有 `/api/*` 请求（除 `/api/auth/*`）需要验证 JWT：

```typescript
// 伪代码
function authMiddleware(req) {
  const token = req.cookies.get('token') || req.headers.authorization;
  const payload = jwt.verify(token, SECRET);
  // payload = { enterpriseId, market, premiumRate, currency, role }
  req.enterprise = payload;
}
```

### 3.3 运营后台登录

```
POST /api/auth/admin-login
```

与企业登录类似，但查 admin_users 表，JWT 中 role = "admin"。

### 3.4 路由保护（前端）

| 路由 | 访问权限 |
|------|---------|
| `/login` | 未登录 |
| `/`, `/orders`, `/wallet`, `/settings/*` | 企业用户（已登录） |
| `/admin/*` | 管理员（已登录） |

## 4. 核心业务流程

### 4.1 运营创建企业 → 企业登录下单（完整链路）

```
运营管理员                    系统                         企业用户
    │                         │                             │
    │  POST /api/admin/       │                             │
    │  enterprises            │                             │
    │ {name, phone, password, │                             │
    │  premiumRate, credit..} │                             │
    │ ──────────────────────→ │                             │
    │                         │ 写入 enterprises 表          │
    │                         │ 密码 bcrypt 哈希存储         │
    │    ← 创建成功 ────────  │                             │
    │                         │                             │
    │  把手机号和密码          │                             │
    │  告知企业客户 ─────────────────────────────────────→  │
    │                         │                             │
    │                         │    POST /api/auth/login     │
    │                         │ ← ────────────────────────  │
    │                         │    验证通过，返回 JWT        │
    │                         │ ─────────────────────────→  │
    │                         │                             │
    │                         │    进入下单页面 /            │
    │                         │    选地址、选车型            │
    │                         │                             │
    │                         │    POST /api/quotation      │
    │                         │ ← ────────────────────────  │
    │                         │                             │
    │                         │ ── 调 Lalamove 报价 ──→     │
    │                         │ ←── 返回原价 64.35 ────     │
    │                         │                             │
    │                         │ 计算: 64.35 × 1.15 = 74.00 │
    │                         │ 返回溢价后价格               │
    │                         │ ─────────────────────────→  │
    │                         │                             │
    │                         │    用户确认下单              │
    │                         │    POST /api/orders         │
    │                         │ ← ────────────────────────  │
    │                         │                             │
    │                         │ 1. 检查额度 ≥ 74.00         │
    │                         │ 2. 调 Lalamove 下单          │
    │                         │ 3. 写入 orders 表           │
    │                         │ 4. 写入 credit_transactions │
    │                         │ 5. 更新 used_credit         │
    │                         │ ─────────────────────────→  │
    │                         │    下单成功，跳转订单列表     │
    │                         │                             │
    │  运营后台看到新订单      │                             │
    │ ← ──────────────────── │                             │
    │  LLI=74.00 LLM=64.35   │                             │
    │  利润=9.65              │                             │
```

### 4.2 报价接口（关键接口）

```
POST /api/quotation
```

**Request（前端发）：**
```json
{
  "serviceType": "VAN",
  "stops": [
    { "address": "中环皇后大道中99号", "lat": 22.282, "lng": 114.158 },
    { "address": "尖沙咀广东道28号", "lat": 22.295, "lng": 114.172 }
  ],
  "specialRequests": ["LALABAG"],
  "scheduleAt": null
}
```

**后端逻辑：**
```
1. 从 JWT 拿到 enterprise.market 和 enterprise.premiumRate
2. 组装 Lalamove 请求，加 HMAC 签名
3. POST https://rest.lalamove.com/v3/quotations
4. 拿到 Lalamove 返回:
   {
     quotationId: "1514140994227007571",
     priceBreakdown: { base: "56", total: "64.35", currency: "HKD" },
     distance: { value: "5836", unit: "m" }
   }
5. 计算溢价: 64.35 × 1.15 = 74.00
6. 返回给前端
```

**Response（返回前端）：**
```json
{
  "quotationId": "1514140994227007571",
  "expiresAt": "2026-03-25T07:23:39.00Z",
  "price": {
    "original": 64.35,
    "display": 74.00,
    "currency": "HK$"
  },
  "distance": 5836,
  "priceBreakdown": {
    "base": 64.35,
    "premiumRate": 1.15,
    "total": 74.00
  }
}
```

> **注意**：quotationId 有效期 5 分钟。前端需要在确认页显示倒计时，过期后自动重新报价。

### 4.3 下单接口

```
POST /api/orders
```

**Request:**
```json
{
  "quotationId": "1514140994227007571",
  "sender": {
    "name": "李明",
    "phone": "+85263123456"
  },
  "recipients": [{
    "name": "王芳",
    "phone": "+85298765432",
    "stopId": 1
  }],
  "driverNote": "请走后门",
  "lliAmount": 74.00,
  "llmAmount": 64.35
}
```

**后端逻辑：**
```
1. 从 JWT 拿 enterprise_id
2. 查 enterprises 表，计算剩余额度:
   remaining = credit_limit - used_credit
3. 如果 remaining < lli_amount → 返回 403 "额度不足"
4. 调 Lalamove API: POST /v3/orders
5. 拿到 lalamove_order_id, shareLink, status
6. 写入 orders 表
7. 写入 credit_transactions:
   { type: "payment", amount: -74.00, description: "订单支付" }
8. 更新 enterprises.used_credit += 74.00
9. 返回订单信息
```

**Response:**
```json
{
  "orderId": "内部UUID",
  "lalamoveOrderId": "107900701184",
  "status": "calling_driver",
  "shareLink": "https://share.lalamove.com/?HK1234556",
  "lliAmount": 74.00,
  "createdAt": "2026-03-25T08:00:00Z"
}
```

### 4.4 订单状态同步（Webhook）

```
POST /api/webhook/lalamove
```

Lalamove 主动推送，无需鉴权（但建议验签）。

**收到的事件：**
```json
{
  "eventType": "ORDER_STATUS_CHANGED",
  "data": {
    "orderId": "107900701184",
    "status": "PICKED_UP"
  }
}
```

**后端逻辑：**
```
1. 通过 lalamove_order_id 查到内部订单
2. 映射状态（见下方状态映射表）
3. 更新 orders.status
4. 如果 status = COMPLETED → 更新 completed_at
5. 如果 status = CANCELED：
   a. 更新 cancelled_at
   b. 写入退款流水: { type: "refund", amount: +74.00 }
   c. 更新 enterprises.used_credit -= 74.00
6. 如果 DRIVER_ASSIGNED：
   a. 调 GET /v3/orders/{id}/drivers/{driverId}
   b. 写入 driver_name, driver_phone, driver_plate
```

**状态映射表：**

| Lalamove 状态 | 系统内部状态 | 说明 |
|---|---|---|
| ASSIGNING_DRIVER | calling_driver | 正在匹配司机 |
| ON_GOING | in_transit | 司机已接单，前往取货 |
| PICKED_UP | delivering | 已取货，配送中 |
| COMPLETED | completed | 已完成 |
| CANCELED | cancelled | 已取消 |
| REJECTED | calling_driver | 司机拒绝，重新匹配 |
| EXPIRED | cancelled | 超时无人接单 |

### 4.5 运营后台查询

运营后台的接口只允许 role=admin 的 JWT 访问。

```
GET /api/admin/enterprises          → 企业列表
GET /api/admin/enterprises/:id      → 企业详情（含额度、交易流水）
POST /api/admin/enterprises         → 创建企业
PATCH /api/admin/enterprises/:id    → 修改企业（溢价、额度等）

GET /api/admin/orders               → 全部订单（支持 ?enterprise_id=&status=&country= 筛选）
GET /api/admin/orders/:id           → 订单详情

GET /api/admin/transactions         → 全部交易流水
```

这些接口直接查数据库，不调 Lalamove API。

### 5.3 Lalamove API 代理层

所有 Lalamove 调用统一走一个工具函数：

```typescript
// /lib/lalamove.ts

async function lalamoveRequest(method: string, path: string, body?: object) {
  const timestamp = Date.now().toString();
  const rawSignature = `${timestamp}\r\n${method}\r\n${path}\r\n\r\n${body ? JSON.stringify(body) : ''}`;
  const signature = hmacSHA256(rawSignature, LALAMOVE_SECRET);

  return fetch(`https://rest.lalamove.com${path}`, {
    method,
    headers: {
      'Authorization': `hmac ${LALAMOVE_API_KEY}:${timestamp}:${signature}`,
      'Market': market,  // 从当前企业信息获取
      'Request-ID': crypto.randomUUID(),
      'Content-Type': 'application/json',
    },
    body: body ? JSON.stringify({ data: body }) : undefined,
  });
}
```

## 8. 关键注意事项

1. **报价有效期 5 分钟**：前端确认页需要倒计时，过期自动重新报价
2. **额度检查必须在后端**：前端展示余额仅供参考，真正的扣款校验在下单接口里做
3. **Lalamove Secret Key 绝不能暴露到前端**：所有 Lalamove 调用必须走 Next.js API Routes
4. **Webhook 要幂等**：同一个事件可能推送多次，用 lalamove_order_id + status 做去重
5. **金额精度**：所有金额字段用 decimal，不用 float，避免精度丢失
6. **订单取消要退额度**：取消时写一条 refund 类型的交易流水，并减少 used_credit
7. **中国大陆不在 Lalamove API 覆盖范围**：目前 API 支持 HK/TH/SG/MY/PH/VN/ID/TW/MX/BR/JP
